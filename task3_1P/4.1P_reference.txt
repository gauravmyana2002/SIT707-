Login Form Example

I understood that the login form uses 3 variables -- Username, Password, and Validation Code -- to build a decision table.

Symbol  Meaning        Variable  Test Role

Each combination of symbols forms one rule  one test case. This supports complete and traceable test design.

Part B - Decision Table Testing Using JUnit
In this task, I created JUnit test cases for the LoginForm program by using decision table
testing. The program checks three values: username, password, and validation code.
The idea was to test different combinations of these values and check whether the
result is correct.
For the decision table, I used these symbols:

    ｷ - = empty
    ｷ W = wrong
    ｷ C = correct
    ｷ X = don't care

Here is the decision table I used:

Rule Username Password Validation Code Expected Result

1-    -  X  Login fails

2-    W  X  Login fails

3-    C  X  Login fails

4W    -  X  Login fails

5W    W  X  Login fails

6W    C  X  Login fails

7C    -  X  Login fails

8C    W  X  Login fails

9C    C  -  Validation fails

10 C  C  W  Validation fails
Rule Username Password Validation Code Expected Result

11 C  C  C  Login succeeds

After making the decision table, I wrote the test cases in LoginFormTest.java. I added
the student ID and name in the first two tests, then I added the remaining test cases
based on the table. These included failed login cases, successful login, wrong validation
code, empty validation code, and correct validation code.

Then I ran the test file as a JUnit test in Eclipse. All 12 tests passed successfully.
    ｷ Runs: 12
    ｷ Errors: 0
    ｷ Failures: 0

This shows that the program worked correctly for the tested conditions.
I also ran Main.java and checked the console output. The output showed the failed login
cases, successful login case, and validation code results.

After finishing the task, I uploaded the project to GitHub.
From this task, I understood that decision table testing is useful because it helps
organise different combinations of inputs clearly. JUnit was also useful because it made
it easy to run all the tests and check whether they passed or failed.
